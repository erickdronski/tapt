# Contributing

Thank you for helping improve Tapt.

1. Open an issue before substantial product, schema, or architecture work.
2. Create a focused branch from `main`.
3. Keep real-data provenance, privacy boundaries, and the no-fabrication rule intact.
4. Add or update tests for behavior changes.
5. Run the relevant local validation described in `README.md`.
6. Open a pull request with the problem, approach, validation, risk, and screenshots for UI changes.

Never commit credentials, private production data, copied proprietary media,
or generated content represented as real user or market activity. Security
reports belong in the private process described in `SECURITY.md`.
